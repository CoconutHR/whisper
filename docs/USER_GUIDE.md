# 耳语（whisper）使用说明书

## 1. 设计理念

耳语面向一个固定的四五人小群体，不使用公开房间列表，也不把每项功能包装成卡片。界面以一条竖线划分身份信息和消息正文，保持等宽字体、低干扰配色和稳定的阅读位置。

底部输入区与消息面板使用完全相同的宽度。两者之间只有一条连续分隔线，因此无论窗口多宽，聊天区都像一块完整面板。桌面端消息面板右边界同时作为右侧会话栏的左边界。

## 2. 登录与注册

通过 Go 服务访问页面时，需要登录或注册：

- 个人名称最多 7 个字符。
- 密码至少 1 个字符，不限制字符种类或复杂度。
- `coco` 是系统保留名称，任何用户都不能注册或改名为 `coco`。
- 登录状态保存在 HttpOnly 会话 Cookie 中，服务端只保存 token 哈希；服务更新或重启后会话仍可继续使用，直到 30 天过期或主动退出登录。客户端检测到服务实例变化后会自动刷新页面加载最新资源。

直接打开 `index.html` 时，页面进入本地演示模式，不连接后端。

## 3. 群聊

页面默认进入 `# 公共大厅`。侧栏的“群聊”标题右侧菜单可以新建普通群聊。

- 输入内容后按 Enter 或点击输入框右侧的上箭头发送，Shift+Enter 换行。手机端可以直接使用发送按钮。
- 每个群聊和私聊会分别保留当前页面内尚未发送的文字；切换到其他会话再返回时，输入内容会恢复，发送成功或退出账号后清除。
- 单独发送 `#` 加一个 Emoji（例如 `#😀`）时，消息会隐藏 `#` 并显示为大 Emoji；不加 `#` 时保持普通文字大小。带有其他文字的消息不会触发放大。
- 输入框左侧的笑脸按钮会打开内容小窗，底部可切换 Emoji、表情包和文件。
- 点击群聊消息左侧的昵称，会在输入框中插入 `@昵称`。
- 新建群聊时可以设置群名、群个签、初始成员和“允许新成员查看历史消息”。该选项默认开启，`公共大厅`是系统保留名称，不能用于普通群聊。
- 点击群聊顶部名称进入群信息页。群主可以修改群名、群个签、成员和历史权限，并转移群主或解散群聊；普通成员只能查看信息或退出群聊。
- 普通群新成员是否能看到加入前的历史，由群主在成员加入时的群设置决定；修改设置不会追溯改变已有成员。
- 点击昵称不会进入私聊，私聊只能通过右侧好友或成员列表进入。
- 桌面端消息左侧固定分成三部分：时分秒、日期语义、昵称。时间左对齐，昵称右对齐；昵称最多显示前三个字符，超出部分使用省略号，悬浮可查看完整名称。

## 4. 消息时间

消息时间默认显示，也可以在本人配置页面关闭。

- 当天：`18:36:11`
- 昨天：`18:36:11  昨天`
- 前天：`18:36:11  前天`
- 同年更早：`18:36:11  07月10日`
- 跨年：`18:36:11  2025年12月31日`

默认使用简洁时间格式；鼠标悬浮时，当前时间文本会立即切换为完整时间，例如：

`2026年07月17日 18:36:11.003`

在本人配置中开启“默认显示完整时间”后，上述显示逻辑反转：默认显示完整时间，鼠标悬浮时切换为简洁时间。

手机端把昵称放在消息元信息行左侧、时间放在右侧；昵称仍最多显示前三个字符，超出部分显示省略号。

## 5. Emoji、表情包与文件

- Emoji：点击后插入输入框当前光标位置，可以继续输入并和文字一起发送。
- 表情包：新账户默认没有表情包，页签中只显示添加按钮。可以选择本机的 GIF、JPEG、PNG 或 WebP 图片加入自己的表情包，单张最大 10 MB，最多收藏 120 张。
- 点击自己的表情包后会立即作为一条独立消息发送，并自动关闭小窗。右键点击或在手机上长按别人发送的图片、表情包，可以收藏到自己的表情包；在表情包页右键或长按自己的图片可以移除收藏。移除只影响自己的列表，不会删除历史消息中的图片。
- 文件：切换到文件页签时打开系统文件选择窗口；也可以把文件从系统文件夹拖到输入区。选中文件或松开拖入后，以草稿卡片加入当前会话输入区。
- 在输入框中粘贴剪贴板图片，会直接把图片加入附件草稿；普通文字粘贴不受影响。
- 文件草稿可以删除，也可以再次打开文件页签继续添加。点击发送后才开始上传；上传失败会保留卡片，可以直接重试。
- 单文件最大 50 MB，每条消息最多 5 个文件，合计最大 100 MB。
- GIF、JPEG、PNG 和 WebP 图片不超过 10 MB 时直接显示图片；更大的图片以文件卡片显示，点击查看后才加载原图。图片查看器支持上一张、下一张、下载、滚轮或按钮缩放以及拖动放大后的图片。
- PDF 和 TXT 文件可以从文件卡片的打开按钮或更多菜单在新标签页查看。Word、Excel 和 PowerPoint 文件使用浏览器自身的打开能力；能否直接预览取决于浏览器和系统配置，无法预览时浏览器会下载文件。本应用不会把私有文件发送给第三方在线预览服务。
- MP3、M4A、AAC、WAV、Ogg、WebM 和 FLAC 音频使用站内播放器，提供播放/暂停、进度、音量、静音和倍速控制。
- MP4、WebM、Ogg、MOV 和 M4V 视频使用站内播放器，提供播放/暂停、进度、音量、静音、倍速、画中画和全屏控制，不显示浏览器自带控制栏。
- 音视频能否播放还取决于文件内部使用的编码。页面会先检查当前浏览器的格式支持，并在实际解码失败时自动改为普通文件卡片，文件仍可下载、转发或撤回。开始播放新的音视频会暂停前一个播放器。
- 文件卡片和媒体的更多菜单支持下载和转发。同一个 R2 对象会被目标消息引用，不会为转发重复占用存储空间。
- 自己发送的消息在两分钟内可以从媒体更多菜单撤回。撤回后双方或群成员看到“消息已撤回”，原消息不再授予附件下载权限；仍被其他转发消息或个人表情包引用的对象不会被删除。

文件上传和自定义表情包仅在管理员已配置 Cloudflare R2 时可用。未配置时不影响文字和 Emoji，历史消息中已有的旧版内置表情仍能显示。管理员配置见 [`R2_SETUP.md`](R2_SETUP.md)。

## 6. 好友与成员

右侧栏只保留以下内容：

1. 固定侧栏
2. 群聊列表和新建群聊菜单
3. 好友列表
4. 全部成员

成员列表始终显示所有人：在线成员在前，离线成员变灰并排在后面。点击离线成员仍可留言，消息显示为“待送达”；对方上线后，服务会完成投递并更新状态。

与某人产生私聊记录后，对方会自动进入好友列表。好友名称支持：

- 左键：进入私聊。
- 右键：打开好友菜单，可设置该好友的消息文字颜色、清空消息记录或删除好友。
- 移动端：点击名称右侧的 `⋮` 打开相同菜单。

收到非当前会话的消息，或停留在当前会话但网页已不活跃时，会话名称会短暂轻抖并显示未读数量。程序同时使用页面可见性、窗口焦点、页面生命周期和指针进出状态判断网页是否活跃，以兼容不会正确更新 `document.hidden` 的内置浏览器。连续消息会累加数量并再次轻抖；返回网页或进入对应会话后未读数量自动清除。已读位置保存在服务端，刷新、重新登录或连接中断后仍会恢复；同一账号在其他标签页读过消息后，当前标签页也会同步清除。侧栏收起时，侧栏按钮显示全部会话的未读总数。系统启用“减少动态效果”时只显示未读数量。收到陌生成员的私聊消息后，该成员会自动加入好友列表。

有未读消息时，浏览器标签页标题会以“【新消息】”前缀横向循环滚动；查看未读会话后恢复原标题。

管理员配置 Web Push 后，本人配置页会显示“系统消息通知”开关。首次开启时，页面会先说明浏览器即将弹出权限申请；继续后需要在浏览器弹窗中选择“允许”。每台设备和每个浏览器需要分别开启。页面位于后台、休眠或关闭时，Service Worker 会显示系统通知；点击通知会返回对应会话。Service Worker 会根据当前部署目录自动选择作用域，因此同时支持域名根目录和 `/w/` 等反向代理路径。浏览器权限被拒绝后，需要先在浏览器的网站权限中重新允许，返回页面即可再次勾选重试。

Windows 浏览器会从应用自身加载 Noto COLRv1 字体统一显示 Emoji，并优化中英文界面的系统字体回退；手机和其他桌面系统继续使用各自的原生 Emoji。字体采用 SIL Open Font License 1.1，许可证随字体文件保存在 `assets/fonts/`。

清空记录只影响当前用户看到的会话，不会删除对方的记录。

消息颜色提供默认、绿色、蓝色、青色、琥珀色和玫红色六种选择。它只影响当前用户看到的该好友消息正文，不改变昵称、自己的消息，也不会影响其他用户。颜色偏好会随账户保存，并针对不同配色方案自动调整明暗。

## 7. 本人配置

点击成员列表中的“本人”，进入个人配置页面。本人不是可发送消息的聊天对象，因此配置项集中放在这里：

- 修改个人名称
- 编辑个性签名
- 修改登录密码
- 显示或隐藏消息时间
- 默认显示完整时间（默认关闭）
- 开启或关闭数学公式解析
- 开启或关闭当前设备的系统消息通知（需要管理员配置 Web Push）
- 选择配色方案
- 清空自己的群聊记录

个性签名显示在在线状态右侧，支持 emoji、裸链接和 Markdown 链接：

```text
今天开工 🚀 https://example.com
[项目文档](https://example.com/docs)
```

裸链接显示为蓝色完整网址；Markdown 链接只显示方括号中的文字，并以蓝色链接呈现。

## 8. coco

`@coco` 是永久在线的系统角色，使用蓝色显示，以区别普通用户。

- 实际名称始终为 `coco`，侧栏显示为 `@coco`。
- 每位用户都有独立的 coco 会话。
- 发给 coco 的内容只对当前用户可见，其他成员无法读取。
- coco 不能下线、不能删除，也不能被注册为普通用户名。

coco 可以作为个人的“文件传输助手”式会话，用于保存临时文本、链接和之后扩展的文件消息。

## 9. 数据与备份

Go 服务把账户、密码哈希、好友关系、消息、附件元数据和设置保存到 SQLite 文件 `data/whisper.db`。文件正文不进入 SQLite 或本机数据目录；启用上传后，它们保存在私有 Cloudflare R2 Bucket。

用户信息和明文密码另存到 `data/users-backup.json`。该文件权限为 `0600`，没有加密，不能上传到 Git、发送给他人或存放在公开位置。

备份时应停止服务，并同时复制 `whisper.db` 和 `users-backup.json`。旧的 `state.json` 不再读取，也不会自动迁移。
